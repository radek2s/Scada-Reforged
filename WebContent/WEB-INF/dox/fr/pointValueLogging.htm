Scada-LTS supporte quatre types d'enregistrement de la valeur d'un point

Quand la valeur change qui est le paramètre par défaut. La valeur et son occurence de temps sont enregistrés dans la base seulement si la valeur su point change. Ce règlage offre le meilleur rapport qualité (de l'historique)/espace de stockage. Pour les valeurs numériques, une Tolerance, ou "zone morte", peut également être définie; la valeur sera enregistrée si la différence entre la valeur actuelle et la valeur précédente est supérieure ou égale à la tolérance définie.

Le paramètre Toutes les données provoque l'enregistrement dans la base de données de toutes les mises à jour des données. Utilisez ce paramètre uniquement lorsque vos contraintes l'exigent car celà va entrainer rapidement une grande accumulation de données et ralentira très sérieusement les performances.

Le paramètre Ne pas enregistrer empêche tout enregistrement de ce point dans la base. Notez que dans ce cas vous ne pourrez pas appliquer de rendu graphique pour ce point. Utilisez "Ne pas enregistrer" lorsque vous etes certain de ne pas avoir besoin de l'historique des données.

Le paramètre Intervalle le nombre d'accès à la valeur du point sont différents du nombre d'enregistrement dans la base. Utilisez ce paramètre lorsque des données doivent être fréquemment consultées (par exemple pour des détections d'évènement), mais qu'il n'est pas obligatoire de toutes les enregistrer. Type valeur vous permet de choisir la valeur qui ser enregistrée. "Instant" la valeur actuelle du point, les autres options sont des statistiques sur la valeur du point durant la période définie.

Le paramètre au changement de timestamp. Comparable au paramètre Quand la valeur change, mais c'est l'horodatage qui est comparé et non la valeur.

Rejeter les valeurs extêmes est utilisé pour les appareils pouvant occasionnellemnt diffusée des valeurs erronées. Lorsque Rejeter les valeurs extêmes est sélectionné, les boites de dialogues rejeter limite basse et rejeter limite haute sont activées. Toutes les valeurs dèjà enregistrées seront conservées.

The Purge strategy option allows user to choose strategy of data purging. Two types are supported: Period and Values Limit. Le processus de purge se lance chaque jour à 3h 05 heure locale.

Le paramètre Period determine la durée d'historisation de la valeur du point dans la base de donnée. Les valeurs peuvent en pratique être conservées aussi longtemps que nécessaire, mais notez que ceci peut engendrer des dégradation de performance du système si l'historique d'un point devient trop important. (En fait, la performance du système dépent en grande partie du matériel hôte).

When the Values Limit type is selected, every time purging is performed for datapoint, number of values set in Values Limit will be left in database for this datapoint. The default value can be set in system settings (Values limit for purge).

Taille par défaut du cache est le nombre de valeur qui seront enregistrées en mémoire. Ce paramètre est important quand une méthode autre que "Toutes les données" est choisie. Les valeurs en mémoire seront éventuellement comparées aux valeurs enregistrées dans la base. Une valeur élevée pour la taille du cache peut améliorer les performances du système (car cela peut empêcher les requêtes sur la base), mais la mémoire est alors consommée par le cache, et des problèmes liés au manque de mémoire peuvent survenir. "Vider le cache" n'est pas nécessaire en théorie, mais peut être utile en cas de suppression manuelle de point (en utilisant les fonctions SQL par exemple).
